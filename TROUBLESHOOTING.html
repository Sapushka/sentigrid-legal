Troubleshooting

Read this first

  • Open the app’s Status panel (bottom of the window). It lists key loads, provider requests, and any errors (network, rate-limit, invalid key, etc.).
  • At the top of the Chart card, check the Data source line to see which provider/timeframe rendered (e.g., “Twelve Data (H4)” or “Alpha Vantage (Daily)”).

Common messages

  • “No Twelve Data or Alpha Vantage keys found.” — Enter keys in API Keys → Save Keys → Refresh.
  • “Alpha Vantage has no ‘H4’ intraday…” — Use H1 or D, or rely on Twelve Data for H4.
  • “COT is disabled.” — Tick Enable COT. COT is weekly and some markets can be delayed.
  • “COT: stale … ignored” — Latest COT report is >35 days old; the app hides it to avoid confusion.

If the chart is blank

  1. Keys — In API Keys, click Test Load Keys. If needed, Delete Saved Keys, re-enter, then Save Keys and Refresh.
  2. Timeframe — Switch between D, H4, H1 and press Refresh.
  3. Symbol format — Try EUR/USD and EURUSD, then Refresh.
  4. Provider throttling — Free tiers can rate-limit. Wait a few minutes or add the other provider key to enable fallback.
  5. Closed markets / gaps — Weekends/holidays or provider gaps may show no bars for a period.

Reset (non-destructive)

  • Use Delete Saved Keys in API Keys to clear stored credentials, then re-enter and Save Keys.
  • If you tweaked many inputs, restart the app to return to defaults.

Advanced (optional)

  • Open DevTools: Ctrl+Shift+I (macOS: Cmd+Option+I). Check Console for network/API errors.
  • Consider adding a second provider key to reduce downtime from rate limits.
